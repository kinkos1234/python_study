# 튜토리얼 1: 모델의 작동 원리 (How Models Work)

> 원본: [Kaggle - How Models Work](https://www.kaggle.com/code/dansbecker/how-models-work)

---

## 🎯 학습 목표

머신러닝 모델이 어떻게 작동하는지 개념적으로 이해한다.

---

## 📖 내용 정리

### 배경 시나리오

이 강좌에서 우리는 다음과 같은 시나리오를 배경으로 모델을 만들 것입니다:

> 당신의 사촌은 부동산 투기로 수백만 달러를 벌었습니다. 그는 당신이 데이터 사이언스에 관심이 있다는 것을 알고 함께 사업을 하자고 제안합니다. 그는 자금을 대고, 당신은 **각 집의 가치를 예측하는 모델**을 만드는 것입니다.

사촌에게 과거에 어떻게 예측했냐고 물어보니, 그저 "직감"이라고 합니다. 하지만 더 물어보면, 그가 이전에 본 집들의 **가격 패턴**을 파악하고, 그 패턴으로 새로운 집의 가격을 예측했다는 것을 알 수 있습니다.

머신러닝도 정확히 같은 방식으로 작동합니다.

---

### 결정 트리 (Decision Tree)

우리는 **결정 트리(Decision Tree)** 라는 모델부터 시작합니다. 더 정확한 모델들도 있지만, 결정 트리는 이해하기 쉽고 최고의 모델들의 기본 구성 요소이기도 합니다.

가장 단순한 결정 트리를 예로 들면:

```
집에 침실이 2개보다 많은가?
        |
   아니오 ─────── 예
    |               |
 $178,000       $188,000
```

이 트리는 집을 단 두 카테고리로 나눕니다. 예측 가격은 해당 카테고리에 속하는 집들의 **역사적 평균 가격**입니다.

---

### 핵심 용어

| 용어            | 원문               | 설명                                      |
| --------------- | ------------------ | ----------------------------------------- |
| **학습(훈련)**  | Fitting / Training | 데이터에서 패턴을 포착하는 과정           |
| **훈련 데이터** | Training Data      | 모델을 학습시키는 데 사용하는 데이터      |
| **예측**        | Predict            | 학습된 모델을 새로운 데이터에 적용하는 것 |
| **리프**        | Leaf               | 트리에서 최종 예측값이 있는 끝 노드       |

---

### 결정 트리 개선하기

어떤 결정 트리가 더 나을까요?

- **트리 1**: 침실 수에 따라 나눔 → 실제 가격에 더 잘 반영됨 ✅
- **트리 2**: 의미 없는 기준으로 나눔 → 덜 유용 ❌

침실 수가 많을수록 집값이 높다는 현실을 반영한 트리 1이 더 합리적입니다.

그러나 이 모델은 욕실 수, 대지 면적, 위치 등 많은 요소를 반영하지 못합니다. 더 많은 요소를 반영하려면 **더 깊은 트리(deeper tree)** 를 사용합니다.

```
집에 침실이 2개보다 많은가?
        |
   아니오            예
    |                |
대지 1,500㎡ 이상?  대지 1,500㎡ 이상?
  |       |          |         |
$146k   $188k      $170k     $233k
```

예측하고 싶은 집의 특성을 따라 트리를 내려가면 **리프(leaf)** 에 도달하고, 그곳의 값이 예측 가격입니다.

---

## 💡 핵심 정리

- 머신러닝은 데이터에서 **패턴을 학습**하고 그 패턴으로 **예측**합니다
- **결정 트리**는 가장 기본적인 ML 모델로, 질문들을 통해 데이터를 분류합니다
- 트리가 **깊을수록** 더 많은 특성을 반영할 수 있습니다
- 데이터로 모델을 학습시키는 과정을 **fitting(훈련)** 이라고 합니다
